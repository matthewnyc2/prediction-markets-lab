§2Terminal states — observable success conditions

TS-01

Strategy trades paper live

The person toggles a strategy on with capital assigned, and within one poll cycle a paper-fill with that strategy as source appears on the ledger.

TS-02

Backtest produces scored results

A strategy + platforms + date-range + bankroll produces a backtest-result with final-equity, Sharpe, Brier, win-rate, and an equity-curve. Zero outbound calls to live trading endpoints during the run.

TS-03

Cross-platform / cross-time compare

≥2 comparison-runs covering ≥2 platforms or ≥2 date-ranges render together on one overlay-P&L chart plus a comparison-table with full metrics per run.

§6Semantic spec highlights

Key invariants

  • INV-001 equity = bankroll + realised-pnl + unrealised-pnl at all times
  • INV-002 market-price ∈ [0, 1] always
  • INV-006 backtest reads only stored-price-history, never live endpoints
  • INV-007 no code path writes to a real-money trading endpoint
  • INV-008 Σ assigned-capital of ON strategies ≤ equity

Forbidden states

  • FS-001 strategy ON with assigned-capital = 0
  • FS-003 completed backtest with null results
  • FS-005 comparison-run referencing a deleted backtest-id
  • FS-006 adapter HEALTHY but last-sync > 10× poll interval
  • FS-007 new paper-fill with source = a strategy whose state is OFF
