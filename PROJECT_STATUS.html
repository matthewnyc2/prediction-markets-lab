What works right now

You can run a backtest today

Open a terminal in the project folder and type this:

PYTHONPATH=src python -m predmarkets.cli backtest \ --strategy kelly --platform mock \ --bankroll 10000 --markets 3 --ticks 40

You'll get a full scored result — final equity, Sharpe ratio, Brier score, max drawdown, win rate, number of trades. The markets are fake (fabricated by a "mock" adapter), but the math and the replay engine are real.

The math is proven correct

Every metric formula is tested:

  • Brier score — how well-calibrated a probability forecaster is
  • Sharpe ratio — risk-adjusted return
  • Sortino ratio — downside-adjusted return
  • Max drawdown — worst peak-to-trough decline
  • Win rate — fraction of profitable trades
  • Kelly fraction — optimal bet size given edge

44 tests cover the edge cases (empty input, zero variance, extreme prices, cancelled markets, ties, etc.). All green.

Two known bugs to fix next

① Kelly strategy over-trades and goes bankrupt

What's happening: the Kelly strategy re-calculates a fresh bet on every market tick. When there's a consistent edge, it keeps buying and buying until the bankroll is spent. In the sample run, it blew $10,000 across 19 trades before any market resolved.

Why it's easy to fix: one-line change — track existing position per market, skip if already holding. The math isn't wrong; the timing is wrong.

BUG file: src/predmarkets/strategies/kelly_sizing.py

② Drawdown can show > 100%

What's happening: because of bug #1, the account goes briefly negative before the "bankrupt" check fires, producing an impossible 104% drawdown number.

Why it's easy to fix: goes away automatically once bug #1 is fixed. If we want belt-and-braces, a one-line clamp at zero does it.

BUG file: src/predmarkets/engine/backtester.py
