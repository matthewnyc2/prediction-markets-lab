How this works

Plain English for non-programmers.

THE DATA

120 real resolved markets

A Python script hit Manifold Markets' public API and pulled 120 markets that have already been resolved yes/no. For each one, it collected the complete history of every bet placed on it, and reconstructed the price over time from those bets.

THE ENGINE

Plays each market back, bet by bet

For every strategy, the engine replays each market's history in order. At each moment in time, the strategy decides whether to buy YES or NO. If it buys, the engine tracks the position until the market resolves, then credits the win or the loss.

THE PORT

Runs entirely in your browser

The Python engine was ported line-for-line to JavaScript, so everything you're seeing runs on your computer, not a server. Pick a strategy, see the result in milliseconds, tweak the settings, see the new result.

THE TESTS

53 unit tests in the Python source

Every math formula (how to compute profit, risk, volatility, etc.) is tested against edge cases. The browser version is a translation of that tested Python code — same inputs, same outputs.

⚠ Honest caveats

  • Manifold uses play-money (mana). The markets are real, the bets are real, the outcomes are real — but the currency is virtual. On real-money platforms (Kalshi, Polymarket) the dynamics would differ.
  • Past performance ≠ future results. A strategy that made money across these 120 markets might lose across a different set of markets or a future time period. This is decision support, not a crystal ball.
  • The test is deliberately naive. No transaction fees, no tax, no liquidity constraints beyond a simple slippage model. Real trading is harsher.
  • Nothing here touches a real trading account. The engine has no code path that submits real orders — by construction.
