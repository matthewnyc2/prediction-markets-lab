04

What the numbers mean

Quant metrics, explained without jargon.

Sharpe ratio

Risk-adjusted return. > 1 is good, > 2 is rare. Penalizes volatility — a bumpy 10% year scores lower than a smooth 8% year.

Brier score

How well-calibrated your bets' implied forecasts are. 0 = perfect, 0.25 = coin-flip, 1 = always confidently wrong. Lower is better.

Max drawdown

Largest peak-to-trough decline during the run. A 30% drawdown means at the worst point you were down 30% from your high-water mark.

Win rate

Fraction of closed positions that made money. High win rate with losing PnL = small wins, big losses.

Kelly fraction

Optimal bet size given your edge. Full Kelly maximises growth but is volatile; quarter-Kelly is the industry-standard conservative choice.

Bankrupt

Whether the strategy spent its entire bankroll during the run. A hard failure that should trigger "redesign", not "tweak parameters".

05

How this was built

Spec-first, test-driven, real-data-backed.
DATA

Real Manifold bets, reconstructed

A Python script hits api.manifold.markets/v0, collects resolved binary markets with ≥5 bettors and ≥50 mana volume, pulls full bet history, and rebuilds the price-over-time from each bet's probBefore / probAfter fields.

ENGINE

Event-driven backtester

Same code path for backtesting and live paper-trading. 53 unit tests cover every metric edge case. Slippage model adjusted for Manifold's constant-function market maker.

PORT

Python → JavaScript, line-for-line

The browser engine is a faithful port: same metric formulas, same portfolio bookkeeping, same deterministic PRNG. Given the same seed + dataset, both runtimes produce identical results.

UI

Static site, zero build step

3 JS files + 1 CSS + 1 HTML + 1 JSON dataset. Served directly from /docs on GitHub Pages. No bundler, no Node server, no CDN dependency beyond Chart.js.

Honest caveats

  • Manifold uses play-money (mana). Platform real, markets real, bets real — but currency is virtual. Incentives differ from real-money platforms like Kalshi and Polymarket.
  • Backtest overfitting is real. A green result here doesn't prove a strategy works elsewhere. Out-of-sample validation, walk-forward testing, and statistical significance matter.
  • Slippage is simplified. The model uses a flat 1¢ base for Manifold's CFMM. Real slippage depends on liquidity pool size.
  • Dataset skew. These 120 markets are a recent-resolved sample with modest volume filters. Results would change on a different slice.
  • No real-money trading. The engine has no write path to any trading API — by construction.
