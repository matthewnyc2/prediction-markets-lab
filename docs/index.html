What the numbers mean

Quant metrics, explained plainly.

Sharpe ratio

Risk-adjusted return. > 1 is good, > 2 is rare. Penalizes volatility — a strategy that returns 10% with bumpy swings scores lower than one that returns 8% smoothly.

Brier score

How well-calibrated your bets' implied forecasts are. 0 = perfect, 0.25 = coin-flip, 1 = always confidently wrong. Lower is better. Missing if no positions closed.

Max drawdown

Largest peak-to-trough decline during the run. A 30% drawdown means at the worst point you were down 30% from your high-water mark.

Win rate

Fraction of closed positions that made money. High win rate with losing PnL = small wins, big losses. Low win rate with winning PnL = rare but huge wins.

Kelly fraction

Optimal bet size given your edge. Full Kelly maximises growth but has high variance; quarter-Kelly is the industry-standard conservative choice.

Bankrupt

Whether the strategy spent its entire bankroll during the run. A hard failure — the kind of event that should trigger "redesign", not "tweak parameters".

How this was built

Spec-first, test-driven, then ported to the browser.
  • PHASE 1 · DESIGN

    Spec locked before any code

    Wrote a complete narrative spec, disambiguated 25 entities, declared 3 observable success conditions, picked the stack, selected a design system, generated wireframes. All frozen.

  • PHASE 2 · REVIEW

    297 rules, 99 contracts, 1 proof

    Extracted 297 rules from the design, mapped 99 user interactions to function contracts, derived a backward state chain, and verified chain correctness via cycle detection + reachability proofs.

  • PHASE 3 · BUILD

    Python engine + 53 unit tests

    Event-driven backtest engine, pluggable strategy ABC, pure-function metrics. Every formula tested against edge cases (empty input, zero variance, cancelled markets, extreme prices).

  • PIVOT

    Shipped a browser port

    The Python engine was ported line-for-line to JavaScript, so the backtester runs client-side with no server. Same logic, same metrics, same deterministic seed → identical results.

Honest caveats

This is a portfolio demo, not a get-rich device. What you should know:

  • Markets are synthetic. The mock adapter generates deterministic random-walks that resolve according to a configurable true probability. They have the right shape (prices in [0, 1], drift, noise, resolution) but are not real Kalshi/Polymarket/Manifold data.
  • Backtest overfitting is real. A green backtest does not mean a strategy will work live. Over 50 years of quant finance has shown that most in-sample winners decay out-of-sample.
  • Slippage is simplified. The model is mid ± 1¢ with tier-based book walking. Real markets are harsher (fees, adverse selection, latency).
  • No real-money trading. The engine has no write path to any trading API — by construction. This project cannot place a real trade, no matter what.
  • Use for decision support, not decisions. The tool forces quant-thinking discipline on gut-feel intuitions. That's genuinely valuable. It does not replace out-of-sample validation or statistical rigor.
