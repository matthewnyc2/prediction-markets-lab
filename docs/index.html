What the numbers mean

Quant metrics, explained plainly.

Sharpe ratio

Risk-adjusted return. > 1 is good, > 2 is rare. Penalizes volatility — a strategy that returns 10% with bumpy swings scores lower than one that returns 8% smoothly.

Brier score

How well-calibrated your bets' implied forecasts are. 0 = perfect, 0.25 = coin-flip, 1 = always confidently wrong. Lower is better.

Max drawdown

Largest peak-to-trough decline during the run. A 30% drawdown means at the worst point you were down 30% from your high-water mark.

Win rate

Fraction of closed positions that made money. High win rate with losing PnL = small wins, big losses.

Kelly fraction

Optimal bet size given your edge. Full Kelly maximises growth but has high variance; quarter-Kelly is the industry-standard conservative choice.

Bankrupt

Whether the strategy spent its entire bankroll during the run. A hard failure — the kind of event that should trigger "redesign", not "tweak parameters".

How this was built

Spec-first, test-driven, real-data-backed.
  • DATA

    Real Manifold bets, reconstructed into price series

    Python script (scripts/fetch_manifold.py) hits api.manifold.markets/v0, collects resolved binary markets with ≥5 unique bettors and ≥50 mana volume, pulls the full bet history, and rebuilds the price-over-time from each bet's probBefore/probAfter fields.

  • ENGINE

    Event-driven backtester + 6 pure metrics

    Same code path for backtesting and live paper-trading. 53 unit tests cover every metric edge case. Slippage model walks the order book on oversized bets. No dependencies on network, filesystem, or clock in the hot path.

  • PORT

    Python → JavaScript, line-for-line

    The browser engine is a faithful port of the Python source: same PRNG, same slippage model, same metric formulas. A given seed + dataset produces identical results in both runtimes.

  • UI

    Zero-build static site, Chart.js, dark-mode-native

    Design tokens from a Linear-inspired system. Entire app is 3 JS files + 1 CSS + 1 HTML + 1 JSON dataset. Deployed via GitHub Pages directly from /docs.

Honest caveats

This is a portfolio demo, not a get-rich device. What you should know:

  • Manifold uses play-money (mana). The platform is real, the markets are real, the bets are real — but the currency is virtual. Incentives differ from real-money platforms like Kalshi and Polymarket.
  • Backtest overfitting is real. A green backtest here does not prove your strategy will work elsewhere. Out-of-sample validation, walk-forward testing, and statistical significance matter.
  • Slippage is simplified. Real CFMM slippage on Manifold depends on liquidity pool size; this engine uses mid ± 1¢ with tier-based walking as a first-order approximation.
  • Dataset skew. These 120 markets are a sample of recently-resolved markets with modest volume filters. Results would change on a different slice.
  • No real-money trading. The engine has no write path to any trading API — by construction.
